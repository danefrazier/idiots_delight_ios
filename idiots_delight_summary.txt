Idiot's Delight iOS — Build Summary
=====================================
Date: 2026-04-16

This document summarises all features built and decisions made during
the iOS port of Idiot's Delight (originally a Python CLI game).

---

Session 1 — Python CLI fixes (repo: idiots_delight)
----------------------------------------------------
Before porting, the Python version was fully repaired:
  - Broken shuffle replaced with correct random.shuffle().
  - Recursive game loop replaced with while loops (stack overflow fix).
  - Removal rule enforced: lowest showing same-suit card only.
  - Win condition added: four stacks each with exactly one ace.
  - None cards in final round fixed (drawOne() return checked).
  - Ace move validation added.

---

Session 2 — iOS Project Bootstrap
-----------------------------------
New repo: idiots_delight_ios
Xcode 26 / Swift 6 project created targeting iOS 26+.
Key framework decisions:
  - @Observable macro (not ObservableObject) — required by Swift 6
    with SWIFT_DEFAULT_ACTOR_ISOLATION = MainActor.
  - @Bindable for GameView bindings into GameState.
  - PBXFileSystemSynchronizedRootGroup: files auto-discovered, no
    manual "Add Files" step needed (caused "Multiple commands produce"
    build error when attempted).
  - Programmatic cards via SF Symbols + Text (no external card assets).

Core models built:
  Card.swift      — Suit, CardValue enums; Card struct; numericValue;
                    isAce; displayString; suit color/symbol.
  Deck.swift      — 52-card build, triple shuffle, drawOne(), count.
  GameState.swift — All game logic: canRemove, tapStack, dealRound,
                    performAceMove, checkWin, checkAvailableMoves.

Core views built:
  ContentView.swift     — root @State GameState.
  GameView.swift        — header, stacks row, status, deal/new-game
                          buttons.
  StackColumnView.swift — single column; shows top card or empty slot.
  CardView.swift        — rounded rect with suit symbol, value text,
                          red/black colouring, selected highlight.
  EmptyStackView.swift  — dashed border placeholder; .contentShape
                          (Rectangle()) for tap detection.

Session 3 — Rule and UX Fixes
-------------------------------
Ace move restricted to empty stacks only (was: any stack).
New Game button added (bottom of portrait layout).
No-moves detection: auto game-over only when deck is also empty;
  silently waits if cards remain in deck (no alert).

---

Session 4 — Hint Mode
-----------------------
Hints toggle (yellow) added to header.
hintStackIndex computed: prefers optimal move (no advisory); falls
  back to any legal move; suppresses lone-ace hint.
CardView pulsing glow (1.6 s ease-in-out, repeatForever) driven by
  @State pulsing + withAnimation.
Glow stops cleanly on toggle-off via .onChange(of: isHinted) with
  explicit 0.2 s ease-out animation block.

Removal rule relaxed: any card lower than the highest showing same-suit
  card is legal (not strict lowest-only). Advisory returned as a String
  and shown in hint mode only. "Removed X of Y" message always shown.

Session 5 — Game Statistics + Ace Killer
------------------------------------------
GameStats.swift: wins, losses, bestGame, aceKillerCount, winsWithAceKiller.

Ace Killer detection: King over same-suit Ace = permanent deadlock.
AceKillerOverlay modal: names the suit, "Keep Playing" / "Start Over".

hadAceKiller flag tracked in stats.

Session 9 — TestFlight Polish (round 1)
------------------------------
Overlay transparency pass — all three end-of-game overlays (Win, Lose,
  Ace Killer) made semi-transparent so cards remain visible beneath:
  - Full-screen veil: black 55% opacity.
  - Text card background: black 35-45% opacity.
  - Text shadows added for legibility over the game board.

Uniform 3-second delay on all end-of-game outcomes:
  - Win: status shows "You win! Four aces." then 3s delay before overlay.
  - Loss (no moves): "No further moves — game over", 3s delay (existing).
  - Loss (deck empty): "No cards left — game over", 3s delay added.
  - Ace Killer alert: 2s delay so king card renders before overlay.
  - All delays guard against phase change (New Game during window).

ContentView restructured: GameView always mounted; Win/Lose overlaid
  via ZStack with .transition(.opacity) fade instead of view replacement.

Ace Killer stats cleanup:
  - Removed aceKillerLosses (never reliably populated at game end).
  - Kept aceKillerCount: fires synchronously at detection time, before
    the async delay, so it records regardless of player choice.

Session 10 — TestFlight Polish (round 2)
-----------------------------------------
Persistent header bar: "Idiot's Delight" title, Hints toggle, Stats,
  and About buttons now always occupy a single top row in both portrait
  and landscape. Landscape left panel trimmed to round info, status
  message, and New Game only.

Landscape cards dynamic sizing: replaced fixed 58×81 card size with
  GeometryReader-driven calculation. Cards fill the width between left

  to 2 seconds. All three overlays are transparent (55% black veil)
  so cards remain visible beneath.

Session 11 — Landscape Layout Fixes
--------------------------------------
Landscape card overflow fixed: stack 3 label was wrapping and stack 4
  labels were clipped due to safe area insets (notch/Dynamic Island
  ~47pt per side) not being subtracted from available width.
  GeometryReader moved back to wrap the full landscape layout so
  geo.safeAreaInsets.leading + trailing can be read and subtracted
  before computing card width. Card sizes use floor() to avoid
  sub-pixel rounding overflow.

Landscape polish pass:
  - Removed duplicate round/deck text from left panel; lastMessage is
    the single status area in landscape.
  - landscapeDealButton: explicit 3-line text + full-width rounded rect
    replaces capsule that became circular when text wrapped.
  - HStack spacing reduced (20→16) and h-padding (20→8) to recover
    unused space on the right edge.
  - Height clamp raised to 90% so cards fill more vertical screen space.

Landscape card sizing root cause fixed:
  - geo.size.width inside a VStack is already safe-area-constrained.
    Subtracting safeAreaInsets.leading + trailing was double-subtracting,
    shrinking cards and leaving unused green space to the right of the
    Deal button. Safe area subtraction removed from card size calculation.
  - Stack 4 label/count wrapping resolved.

Landscape layout redesigned as 6 equal columns:
  - Previous approach of fixed panel widths + calculated remainder was
    brittle and caused Stack 3/4 label wrapping across multiple attempts.
  - New approach: colWidth = floor((geo.size.width - 16) / 6).
    6 columns: status | card1 | card2 | card3 | card4 | deal button.
    cardWidth = colWidth - 8 (4pt breathing room each side).
    HStack spacing = 0; each column carries its own .frame(width: colW).
    All 4 stack columns are guaranteed identical width.
